Frequently Asked Questions

What is HMAC used for?

HMAC (Hash-based Message Authentication Code) is used to verify both the integrity and authenticity of a message. Common use cases include API request signing, webhook verification, and data integrity checks.

Is my secret key secure?

Yes! All HMAC generation happens client-side in your browser. Your secret keys are never sent to our servers or stored anywhere.

Which algorithms are supported?

We support SHA-256, SHA-512, SHA-1, and MD5. SHA-256 is recommended for most security-critical applications.

Do you offer an API?

Yes! Pro users get access to our REST API for programmatic HMAC generation. Perfect for backend integration.
